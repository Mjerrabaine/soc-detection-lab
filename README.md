# SOC Detection Lab – Splunk + Sysmon

## Overview

This project demonstrates the creation of a local Security Operations Center (SOC) lab used to simulate adversarial behavior and detect malicious activity using SIEM technology.

The lab environment uses Windows and Kali Linux virtual machines to simulate attacks and analyze logs collected with Sysmon and ingested into Splunk.

The objective of this project is to develop detection rules, alerts, and incident analysis based on real attack simulations mapped to the MITRE ATT&CK framework.

---

## Lab Architecture

The lab environment consists of two virtual machines:

Attacker Machine:
- Kali Linux

Victim Machine:
- Windows 10

Monitoring Tools:
- Sysmon
- Splunk Free SIEM

Logs generated by Sysmon are ingested into Splunk where detection queries and alerts are created.

## Simulated Attacks

The following adversarial techniques were simulated and detected:

1. Encoded PowerShell Execution (Command and Scripting Interpreter: PowerShell -T1059.001)  
2. Boot or Logon Autostart Execution: Registry Run Keys / Startup Folder (T1059.001)
3. Suspicious Outbound Network Connection (Application Layer Protocol-T1071)

Each attack includes:

- Attack simulation
- Detection query
- MITRE ATT&CK mapping
- Investigation analysis

---

## Detection Engineering

Custom SPL queries were developed to detect malicious behavior using Sysmon logs.
